# AI 윤리 자격·진단·교육·인증 사업 종합 브리프

> 이 문서는 다른 ChatGPT 방, 다른 AI, 개발 AI, 기획자, 디자이너, 마케터가 지금까지 논의된 사업 방향을 빠르게 이해하도록 만든 **마스터 컨텍스트 문서**입니다.

---

## 1. 사업의 핵심 이해

### 1.1 사업 정체성

이 사업은 단순한 “AI 윤리 소개 교육”이 아니다.  
핵심은 **AI 시대에 윤리를 에티켓이 아니라 ESG·보안·신뢰·컴플라이언스의 필수 체계로 전환하는 것**이다.

### 1.2 사업의 4대 축

| 구분 | 사업 축 | 핵심 의미 |
|---|---|---|
| 1 | AI 윤리 인증 | 기업·기관·사업·서비스·콘텐츠·AI 활용체계에 대해 윤리 진단과 컨설팅을 수행하고 인증 또는 등급화 |
| 2 | AI 윤리 교육 | 유치원부터 초중고, 대학생, 직장인, 관리자, 전문가까지 대상별 교육 |
| 3 | AI 윤리 시험 | 민간자격 시험, 급수형 평가, 진단 기반 시험, 실습·사례 기반 평가 |
| 4 | AI 윤리 인프라 | AI 윤리 마인드가 있는 신뢰 가능한 전문가 풀, 강사, 심사관, 컨설턴트, 연구자 네트워크 구축 |

### 1.3 공신력 자산

| 자산 | 활용 방향 | 주의 문구 |
|---|---|---|
| 사단법인 국제통역번역협회와 함께 자격 교육·발급 진행 | 교육·시험·자격 운영 신뢰 기반 | 협회명과 역할은 실제 운영 범위에 맞게 명확히 표기 |
| 민간자격증 발급 가능 체계 | 유치원·초중고 1~9급, 일반 1~3급, 전문 1~2급으로 확장 | 국가공인 자격처럼 오인되지 않게 “민간자격” 명시 |
| 대학 교수 3명과 프롬프트 평가 특허 등록 완료 | 평가모델·시험체계의 전문성 어필 | 특허가 자격 효력이나 법적 인증을 보증한다는 표현은 피함 |
| 공식자료 기반 DB와 글로벌 자료 조사 | 국내외 정책·윤리·규제 근거 기반 사업으로 포지셔닝 | 각 자료의 최신성은 정기 업데이트 필요 |

## 3. 사업 철학과 메시지

### 3.1 핵심 철학

AI 윤리는 “착하게 쓰자”는 캠페인이 아니다.  
앞으로 AI 윤리는 **법·보안·신뢰·ESG·컴플라이언스**와 연결되는 필수 역량이 된다.

### 3.2 홈페이지·사업제안서 공통 핵심 문장

- AI 시대의 경쟁력은 기술이 아니라 신뢰에서 완성됩니다.
- AI 윤리는 에티켓이 아니라 컴플라이언스입니다.
- AI를 활용하는 모든 조직은 AI 윤리 리스크를 진단하고 관리해야 합니다.
- AI를 잘 쓰는 사람보다 중요한 것은 AI를 책임 있게 쓰는 사람입니다.
- AI 윤리는 ESG, 보안, 개인정보, 저작권, 산업 신뢰와 연결되는 새로운 필수 기준입니다.
- 앞으로 기업과 기관은 “AI를 썼는가”보다 “AI를 어떻게 관리했는가”를 증명해야 합니다.

## 4. 자격 체계

### 4.1 대상별 자격 체계

| 대상 | 급수 | 방향 |
|---|---:|---|
| 유치원·초등·중고등 | 1~9급 | AI 안전습관, 개인정보, 딥페이크 경계, AI 사용 예절과 책임 |
| 대학생 | 일반 3급~2급 또는 별도 대학과정 | 과제·논문·리포트·취업·콘텐츠 생성 윤리 |
| 일반 직장인 | 일반 1~3급 | 업무 AI 사용, 보안, 저작권, 검수, 표시, 책임 |
| 전문가·강사·컨설턴트 | 전문 1~2급 | 진단, 교육 설계, 사례 분석, 기업 컨설팅, 리스크 평가 |
| 최고전문가·심사관 | 별도 인증심사관 과정 | 법·규제·컴플라이언스·ESG·기업 인증 심사 |

### 4.2 자격과 진단의 연결

| 단계 | 역할 |
|---|---|
| 무료 진단 | 관심 유도, 위험 인식, 교육 필요성 체감 |
| 일반 유료 진단 | 개인·직장인·대학생의 수준 분석 및 교육 추천 |
| 전문가 진단 | 강사·컨설턴트·관리자 대상 전문역량 평가 |
| 최고전문가 진단 | 법·규제·인증심사관·기업 컨설팅 역량 평가 |
| B2B 조직 진단 | 기업 직원의 AI 윤리 이해도, 부서별 리스크, 교육 필요성 분석 |
| 자녀/초등학생 진단 | 보호자 대상 후킹. “우리 아이는 AI 윤리에 안전한가요?” |

## 5. 진단 테스트 구조

### 5.1 프롬프트 평가 기준에서 확장한 AI 윤리 평가축

첨부된 프롬프트 평가 기준의 핵심 축인 **구체성, 일관성, 진실성, 효율성, 윤리성**을 AI 윤리 진단에 확장한다.

| 프롬프트 평가축 | AI 윤리 진단 확장 |
|---|---|
| 구체성 | AI 사용 목적, 대상, 데이터, 공개범위를 구체적으로 이해하는가 |
| 일관성 | 조직·개인 AI 사용 기준이 일관되는가 |
| 진실성 | AI 결과의 사실성, 출처, 검증 필요성을 이해하는가 |
| 효율성 | AI를 효율적으로 사용하되 검수·책임 절차를 생략하지 않는가 |
| 윤리성 | 편향, 차별, 유해성, 권리침해, 안전 문제를 예방하는가 |

### 5.2 추가 평가축

| 평가축 | 설명 |
|---|---|
| 정확성 | AI 결과가 사실·근거·맥락에 맞는지 검증 |
| 맥락성 | 대상, 연령, 산업, 문화, 국가별 차이를 고려 |
| 투명성 | AI 사용 사실, 생성콘텐츠, 자동화 판단을 고지 |
| 책임성 | 오류·피해·오남용 발생 시 책임 주체와 절차 명확화 |
| 공정성 | 차별, 편향, 배제, 불공정한 자동 판단 예방 |
| 개인정보·보안 | 개인정보, 민감정보, 영업비밀, 내부자료 보호 |
| 저작권·권리보호 | 저작권, 초상권, 음성권, 데이터 권리 검토 |
| 컴플라이언스 | 국내외 법·규제·가이드라인·산업기준 준수 |
| ESG·사회적 영향 | 신뢰, 지속가능성, 사회적 책임과 연결 |

## 6. 교육·커리큘럼 방향

### 6.1 교재는 수정이 필요함

기존 교재와 커리큘럼은 AI 윤리 설명 중심에서 벗어나야 한다.  
앞으로는 **진단→교육→시험→자격→인증→컨설팅**으로 연결되도록 설계해야 한다.

### 6.2 교재 구성 원칙

| 원칙 | 내용 |
|---|---|
| 대상별 | 유치원·초등·중고등·대학생·직장인·전문가별 난이도 구분 |
| 사례 중심 | 국내외 AI 윤리 사고, 딥페이크, 개인정보, 저작권, 채용차별, 보이스클로닝 사례 |
| 공식자료 탐색 | OECD, UNESCO, EU AI Act, NIST, ISO, 한국 AI 기본법 등 직접 확인 실습 |
| 진단 연계 | 각 장마다 진단문항, 체크리스트, 점수화, 개선과제 연결 |
| 시험 연계 | 객관식, OX, 사례형, 보고서형, 실습형 문제 포함 |
| 인증 연계 | 기업·기관이 무엇을 증빙해야 하는지 문서화 |
| 마케팅 연계 | 교육 후 “우리 조직도 진단이 필요하다”는 욕구 발생 |

## 8. 중요한 주의사항

- 민간자격증은 국가공인으로 오인되지 않도록 표기한다.
- 특허 등록 사실은 평가모델의 신뢰자산으로 표현하되, 법적 인증 효력처럼 표현하지 않는다.
- AI 윤리 인증은 자체 인증인지, 협회 인증인지, 제3자 평가인지 구분한다.
- 진단 테스트는 법률 자문을 대체하지 않는다는 고지문을 포함한다.
- 아동 대상 진단은 개인정보 수집 최소화, 보호자 동의, 안전한 표현이 필요하다.
- 기업 진단은 개인정보와 내부기밀 입력을 요구하지 않도록 설계한다.
